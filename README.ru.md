# StealthChat

**Сквозное шифрование для любого веб-мессенджера — сообщения выглядят как обычные предложения на выбранном языке.**

StealthChat — расширение Chrome, которое шифрует сообщения перед отправкой и автоматически расшифровывает входящие. Вместо нечитаемого шифротекста зашифрованные сообщения выглядят как обычные предложения на 7 языках — невидимые для платформ, серверов и любого без расширения.

---

## Как это выглядит

```
Вы набираете:      "Встречаемся в 5 у метро"

Что отправляется   "Алиса твердо описал этих древний бухта вместо февраль.
(русский):          Антон глубоко закрыл других горький каньон помимо май."

Что видит ваш
собеседник
со StealthChat:    "Встречаемся в 5 у метро"
```

Никаких маркеров. Никаких тегов. Никаких подозрительных символов.

## Как это работает

### Пайплайн шифрования

```
                    ОТПРАВКА СООБЩЕНИЯ

  "Привет мир"
       │
       ▼
  ┌─────────────┐
  │  Сжатие     │  deflate-raw (встроенный CompressionStream)
  │             │  только если уменьшает размер
  └──────┬──────┘
         ▼
  ┌─────────────┐
  │ Шифрование  │  AES-256-GCM (8-байтный auth tag)
  │             │  + случайный 12-байтный IV
  └──────┬──────┘
         ▼
  ┌─────────────┐
  │ Упаковка    │  версия + флаги + session ID + IV + шифротекст
  │ (прот. v2)  │  компактный заголовок 16 байт
  └──────┬──────┘
         ▼
  ┌─────────────┐
  │ Кодирование │  байты → предложения (7 языков)
  │ (словарь)   │  5 бит на слово, 8 слов на предложение
  └──────┬──────┘
         ▼
  "Алиса твердо описал этих древний бухта вместо февраль.
   Антон глубоко закрыл других горький каньон помимо май."
```

### Кодирование в предложения

Байты упаковываются в 40-битный поток, затем разбиваются на 8 групп по 5 бит. Каждое 5-битное значение (0–31) выбирает слово из категории с 32 словами:

**Шаблон:** `Имя наречие глагол местоимение прилагательное существительное предлог время.`

7 поддерживаемых языков, каждый с 256 словами (8 категорий × 32 слова):

| Язык | Код | Пример |
|------|-----|--------|
| English | EN | Alice bravely arranged several ancient anchor despite morning. |
| Русский | RU | Алиса твердо описал этих древний бухта вместо февраль. |
| Українська | UK | Оксана твердо описав цих давній якір замість лютий. |
| Deutsch | DE | Annika tapfer ordnete diese antik Anker während Februar. |
| Беларуская | BE | Янка моцна апісаў гэтых старажытны якар замест люты. |
| فارسی | FA | دارا محکم نوشت اینها باستانی لنگر بجای اردیبهشت. |
| Қазақша | KK | Айгүл берік сипаттады бұлар ежелгі зәкір орнына ақпан. |

**Одно предложение = 8 слов = 40 бит = 5 байт данных**

Декодирование автоматически определяет язык по первому слову.

## Установка

1. Скачайте: `git clone` или **Code → Download ZIP**
2. Откройте `chrome://extensions/`
3. Включите **Режим разработчика**
4. **"Загрузить распакованное расширение"** → выберите папку **`extension/`**
5. Закрепите иконку (🧩 → 📌)

> **Важно:** Расширение должно быть установлено у обоих собеседников.

## Использование

### Шаг 1 — Выберите язык и настройте шифрование

1. Нажмите на иконку **StealthChat** (🔒)
2. Выберите язык (**EN / RU / UK / DE / BE / FA / KK**)
3. Нажмите **"Начать шифрование"**
4. Ключ скопируется в буфер (очистится через 30 секунд)
5. Вставьте в чат и отправьте
6. Расширение собеседника распознает ключ и скопирует ответ
7. Собеседник вставляет и отправляет ответ
8. Обе стороны увидят **"Шифрование активно"**

### Шаг 2 — Отправка

1. Введите сообщение как обычно
2. Нажмите **Ctrl+Enter** — текст заменится на закодированные предложения
3. Передумали? Нажмите **Ctrl+Enter** ещё раз — вернётся оригинальный текст
4. Отправьте обычным способом

### Шаг 3 — Получение

Автоматически — StealthChat сканирует страницу и расшифровывает на месте. Рядом с расшифрованными сообщениями появляется 🔒. Язык определяется автоматически.

### Управление

- **Тема** — тёмная/светлая (☀/🌙)
- **Язык** — EN, RU, UK, DE, BE, FA, KK
- **Переключатель** — вкл/выкл шифрование
- **Экспорт/Импорт** — резервное копирование ключей
- **Ротация ключей** — автоматическая (каждые 50 сообщений) + ручная
- **Отпечаток** — сетка hex для верификации собеседника

## Безопасность

| Компонент | Реализация |
|-----------|-----------|
| Обмен ключами | ECDH P-256 (Web Crypto API) |
| Вывод ключа | HKDF-SHA256 |
| Шифрование | AES-256-GCM |
| Целостность | GCM auth tag (8 байт / 64 бита) |
| IV/Nonce | 12 байт, криптографически случайный |
| Сжатие | Встроенный deflate-raw (CompressionStream API) |
| Forward secrecy | Авто-ротация каждые 50 сообщений + ручная HKDF-ratchet |
| Fingerprint | SHA-256 ключа, первые 16 байт |
| Криптобиблиотека | Только Web Crypto API — ноль зависимостей |
| Защита от XSS | HTML-экранирование + DOM API |
| Валидация импорта | Проверка схемы, защита от prototype pollution |
| Буфер обмена | Автоочистка через 30 секунд |

## Бинарный протокол v2

```
┌─────────┬───────┬────────────┬────────────┬──────────────────────┐
│ Версия  │ Флаги │ Session ID │     IV     │    Шифротекст        │
│ 1 байт  │1 байт │  2 байта   │  12 байт   │    N байт            │
│  0x02   │       │            │            │  (включая 8Б тег)    │
└─────────┴───────┴────────────┴────────────┴──────────────────────┘

Флаги: бит 7 = сжатие, биты 0-3 = тип
Тип: 0x01 = зашифрованный текст
     0x02 = запрос обмена ключами
     0x03 = ответ обмена ключами
```
