# StealthChat

**Сквозное шифрование для любого веб-мессенджера — сообщения выглядят как обычные английские предложения.**

StealthChat — расширение Chrome, которое шифрует сообщения перед отправкой и автоматически расшифровывает входящие. Вместо нечитаемого шифротекста зашифрованные сообщения выглядят как обычные английские предложения — невидимые для платформ, серверов и любого без расширения.

---

## Как это выглядит

```
Вы набираете:      "Встречаемся в 5 у метро"

Что отправляется:  "Alice slowly brought a neat desert along December.
                    Grace happily entered every calm ocean beyond Friday."

Что видит ваш
собеседник
со StealthChat:    "Встречаемся в 5 у метро"
```

Никаких маркеров. Никаких тегов. Никаких подозрительных символов. Просто обычные английские предложения.

---

## Как это работает

### Общая схема

```
┌──────────────────────────────────────────────────────────────────┐
│                        ВАШ БРАУЗЕР                               │
│                                                                  │
│  ┌─────────┐    Ctrl+Enter    ┌──────────┐    закодированный    │
│  │  Ваш    │ ──────────────►  │StealthChat│  ──────текст────►  📤│
│  │  текст  │                  │расширение │                     │
│  └─────────┘                  └──────────┘                      │
│                                                                  │
│  ┌─────────┐    расшифровано  ┌──────────┐    закодированный    │
│  │ Чистый  │ ◄──────────────  │StealthChat│  ◄────текст──────  📥│
│  │  текст  │                  │расширение │                     │
│  └─────────┘                  └──────────┘                      │
└──────────────────────────────────────────────────────────────────┘

                    ┌─────────────┐
                    │   Сервер    │  Видит только английские предложения
                    │  платформы  │  ✗ Не может прочитать сообщения
                    └─────────────┘
```

### Пайплайн шифрования

```
                    ОТПРАВКА СООБЩЕНИЯ

  "Привет мир"
       │
       ▼
  ┌─────────────┐
  │  Сжатие     │  UTF-8 → deflate
  │  (pako)     │
  └──────┬──────┘
         ▼
  ┌─────────────┐
  │ Шифрование  │  AES-256-GCM
  │             │  + случайный 12-байтный IV
  └──────┬──────┘
         ▼
  ┌─────────────┐
  │ Упаковка    │  версия + тип + session ID + IV + шифротекст
  │ (протокол)  │
  └──────┬──────┘
         ▼
  ┌─────────────┐
  │ Кодирование │  байты → английские предложения
  │ (словарь)   │  4 бита на слово, 8 слов на предложение
  └──────┬──────┘
         ▼
  "Alice slowly brought the dark clinic behind Thursday.
   Emma carefully caught a bright garden above Monday."
```

### Кодирование в предложения

Каждый байт зашифрованных данных разбивается на два 4-битных нибла. Каждый нибл выбирает слово из определённой категории:

```
Байт: 0xA3 = 1010 0011

     Старший нибл: 1010 = 10 ──► adjectives[10] = "neat"
     Младший нибл: 0011 =  3 ──► nouns[3]       = "desert"
```

**Шаблон предложения:** `Имя наречие глагол артикль прилагательное существительное предлог время.`

| Позиция | Категория | 16 слов | Пример |
|---------|-----------|---------|--------|
| 1 | Имена | Alice, Bob, Carol... | Alice |
| 2 | Наречия | quickly, slowly... | slowly |
| 3 | Глаголы | brought, carried... | brought |
| 4 | Артикли | the, a, one, this... | a |
| 5 | Прилагательные | bright, calm, dark... | neat |
| 6 | Существительные | bridge, castle... | desert |
| 7 | Предлоги | about, above... | along |
| 8 | Слова времени | Monday, Tuesday... | December |

**→ Одно предложение = 8 слов = 32 бита = 4 байта данных**

### Обмен ключами (ECDH)

```
     Алиса                                    Боб
       │                                        │
       │  1. Генерирует пару ключей ECDH        │
       │  2. Кодирует публичный ключ            │
       │     в предложения                      │
       │                                        │
       │──── "Carol eagerly delivered..." ────► │
       │     (содержит публичный ключ Алисы)    │
       │                                        │
       │                    3. Генерирует пару ключей ECDH
       │                    4. Вычисляет общий секрет
       │                    5. Кодирует свой публичный ключ
       │                                        │
       │ ◄─── "David kindly fetched..." ────────│
       │      (содержит публичный ключ Боба)    │
       │                                        │
       │  6. Вычисляет тот же общий секрет      │
       │  7. Выводит ключ AES-256 через HKDF    │
       │                                        │
       │  ════ ЗАШИФРОВАННЫЙ КАНАЛ ════         │
       │  У обоих сторон одинаковый AES ключ    │
       │  Сообщения теперь зашифрованы          │
```

Третья сторона видит оба публичных ключа (в виде предложений), но вычислить общий секрет невозможно — это задача дискретного логарифма на эллиптических кривых.

## Установка

### Chrome / Edge / Brave (любой Chromium-браузер)

**Шаг 1 — Скачайте расширение**

```bash
git clone https://github.com/user/StealthChat.git
```

Или нажмите **Code → Download ZIP** на GitHub и распакуйте в любое место.

**Шаг 2 — Откройте страницу расширений**

Перейдите на страницу расширений в вашем браузере:

| Браузер | Адрес |
|---------|-------|
| Chrome  | `chrome://extensions/` |
| Edge    | `edge://extensions/` |
| Brave   | `brave://extensions/` |

**Шаг 3 — Включите режим разработчика**

```
┌─────────────────────────────────────────────────────────┐
│  Расширения                          [Режим разработчика ☑] │
│                                                         │
│  ┌─────────────────────────────────┐                    │
│  │ Загрузить распакованное расширение │  ← Нажмите      │
│  └─────────────────────────────────┘                    │
└─────────────────────────────────────────────────────────┘
```

Включите переключатель **Режим разработчика** в правом верхнем углу страницы.

**Шаг 4 — Загрузите расширение**

1. Нажмите **"Загрузить распакованное расширение"** (появляется после включения режима разработчика)
2. В диалоге выбора файлов перейдите в скачанный репозиторий
3. Выберите папку **`extension/`** (не корневую папку — именно подпапку `extension/`)
4. Нажмите **"Выбрать папку"**

**Шаг 5 — Закрепите в панели инструментов**

```
┌──────────────────────────┐
│  🧩 Расширения            │
│  ┌────────────────────┐  │
│  │ 🔒 StealthChat  📌 │  ← Нажмите на булавку
│  └────────────────────┘  │
└──────────────────────────┘
```

1. Нажмите иконку пазла (🧩) в панели инструментов браузера
2. Найдите **StealthChat** в списке
3. Нажмите **булавку** (📌), чтобы закрепить иконку в панели

**Готово!** Иконка замка StealthChat теперь видна в панели инструментов.

> **Важно:** Расширение должно быть установлено у обоих собеседников. Поделитесь ссылкой на репозиторий.

## Использование

### Шаг 1 — Настройте шифрование с собеседником

```
  Вы                                          Собеседник
   │                                               │
   │  1. Нажмите 🔒 → "Start Encryption"           │
   │  2. Ключ скопирован в буфер                   │
   │                                               │
   │  3. Вставьте и отправьте ─────────────────►   │
   │     "Carol eagerly delivered the bright       │
   │      castle across Saturday..."               │
   │                                               │
   │                       4. Расширение распознаёт ключ
   │                       5. Ответ скопирован в буфер
   │                                               │
   │  ◄─────────────────── 6. Вставить и отправить │
   │     "David kindly fetched one fair             │
   │      garden behind Sunday..."                 │
   │                                               │
   │  ✅ "Encryption active"    ✅ "Encryption active"
```

1. Откройте любой веб-чат (Telegram Web, WhatsApp Web, Facebook Messenger и т.д.)
2. Нажмите на иконку **StealthChat** (🔒) в панели инструментов
3. Нажмите **"Start Encryption"**
4. Закодированный публичный ключ автоматически скопируется в буфер обмена
5. Вставьте его в чат и отправьте — выглядит как обычные английские предложения
6. Расширение собеседника автоматически распознает ключ и скопирует ответ
7. Собеседник вставляет и отправляет ответ
8. Обе стороны увидят **"Encryption active"**

### Шаг 2 — Отправка зашифрованных сообщений

```
  ┌────────────────────────────────────┐
  │ Поле ввода чата                    │
  │                                    │
  │ "Встречаемся в 5 у метро"         │  ← Пишете как обычно
  │                                    │
  │           Нажмите Ctrl+Enter       │
  │                  ▼                 │
  │ "Alice slowly brought a neat       │  ← Текст заменён
  │  desert along December. Grace      │
  │  happily entered every calm        │
  │  ocean beyond Friday."             │
  │                                    │
  │          [Отправить] ← Отправляете │
  └────────────────────────────────────┘
```

1. Введите сообщение в поле ввода чата как обычно
2. Нажмите **Ctrl+Enter** — текст заменится на закодированные предложения
3. Отправьте сообщение обычным способом (Enter, кнопка "Отправить" и т.д.)

### Шаг 3 — Получение сообщений

```
  ┌─────────────────────────────────────────┐
  │  Собеседник:                            │
  │  "Встречаемся в 5 у метро" 🔒           │  ← Авто-расшифровка
  │                                         │
  │  Без расширения было бы видно:          │
  │  "Alice slowly brought a neat desert    │
  │   along December..."                    │
  └─────────────────────────────────────────┘
```

Автоматически — StealthChat сканирует страницу, находит закодированные сообщения и расшифровывает их на месте. Рядом с расшифрованными сообщениями появляется иконка 🔒.

### Управление сессиями

Нажмите на иконку StealthChat чтобы увидеть:
- **Статус текущей страницы** — зашифровано или нет
- **Переключатель** — включить/выключить шифрование для этой страницы
- **Выбор сессии** — переключение между несколькими сессиями на одной странице
- **Список сессий** — все активные зашифрованные сессии
- **Экспорт/Импорт** — резервное копирование и восстановление ключей
- **Reset All** — удалить все сессии и ключи

### Верификация собеседника (Fingerprint)

```
  ┌──────────────────────────────────┐
  │  🔒 StealthChat           v1.0  │
  │  ● Шифрование активно            │
  │                                  │
  │  FINGERPRINT                  ?  │
  │  ┌──────┬──────┬──────┬──────┐   │
  │  │  a3  │  b5  │  c7  │  d9  │   │
  │  ├──────┼──────┼──────┼──────┤   │
  │  │  e1  │  f2  │  0a  │  4b  │   │
  │  ├──────┼──────┼──────┼──────┤   │
  │  │  8c  │  3e  │  7f  │  1d  │   │
  │  ├──────┼──────┼──────┼──────┤   │
  │  │  2b  │  9a  │  5e  │  0c  │   │
  │  └──────┴──────┴──────┴──────┘   │
  │                                  │
  │  [Rotate Key]  [Reset Keys]      │
  └──────────────────────────────────┘
```

После установки шифрования оба собеседника видят **fingerprint** — сетку 4x4 из hex-пар, полученных из общего ключа. Сравните сетку по другому каналу (звонок, встреча) чтобы убедиться, что обмен ключами не был перехвачен.

### Просмотр оригинального шифротекста

Нажмите на расшифрованное сообщение (или иконку 🔒) чтобы переключиться между расшифрованным текстом и оригинальными закодированными предложениями. Иконка меняется на 🔓 при показе шифротекста. Нажмите снова чтобы вернуться.

### Ротация ключей (Forward Secrecy)

Нажмите **"Rotate Key"** в popup чтобы вывести новый ключ шифрования из текущего. Это обеспечивает forward secrecy — если будущий ключ скомпрометирован, прошлые сообщения останутся защищены.

- Оба пользователя должны ротировать ключ одновременно
- Последние 5 старых ключей сохраняются для расшифровки старых сообщений
- Fingerprint меняется после ротации — верифицируйте снова

## Безопасность

| Компонент | Реализация |
|-----------|-----------|
| Обмен ключами | ECDH P-256 (Web Crypto API) |
| Вывод ключа | HKDF-SHA256 |
| Шифрование | AES-256-GCM |
| Целостность | GCM auth tag (16 байт) |
| IV/Nonce | 12 байт, криптографически случайный |
| Forward secrecy | HKDF-ratchet ротация ключей (ручная) |
| Fingerprint | SHA-256 симметричного ключа, первые 16 байт |
| Криптобиблиотека | Только Web Crypto API — без сторонних библиотек |

### От чего защищает StealthChat

- **Чтение сообщений платформой** — сквозное шифрование, сервер видит только предложения
- **Подмена сообщений** — AES-GCM auth tag обнаруживает модификацию
- **Replay-атаки** — уникальный IV для каждого сообщения
- **Обнаружение паттернов** — сообщения выглядят как обычный текст
- **Компрометация будущего ключа** — ротация ключей / PFS

### От чего НЕ защищает StealthChat

- **Скомпрометированное устройство** — если есть доступ к браузеру, есть доступ к ключам
- **MITM при обмене ключами** — верифицируйте fingerprint по другому каналу
- **Метаданные** — платформа видит кто кому пишет и когда
- **Ограничение длины** — зашифрованный текст длиннее оригинала (~60 символов на 4 байта)

## Бинарный протокол

Каждое сообщение имеет следующую структуру:

```
┌─────────┬──────┬────────────┬────────────┬──────────────────────┐
│ Версия  │ Тип  │ Session ID │     IV     │    Шифротекст        │
│ 1 байт  │1 байт│  4 байта   │  12 байт   │    N байт            │
│  0x01   │      │            │            │  (включая auth tag)  │
└─────────┴──────┴────────────┴────────────┴──────────────────────┘

Тип: 0x01 = зашифрованный текст
     0x02 = запрос обмена ключами
     0x03 = ответ обмена ключами
```
