# StealthChat

**Сквозное шифрование для любого веб-мессенджера — сообщения выглядят как обычные предложения на выбранном языке.**

StealthChat — расширение Chrome, которое шифрует сообщения перед отправкой и автоматически расшифровывает входящие. Вместо нечитаемого шифротекста зашифрованные сообщения выглядят как обычные предложения на английском, русском, украинском или немецком — невидимые для платформ, серверов и любого без расширения.

---

## Как это выглядит

```
Вы набираете:      "Встречаемся в 5 у метро"

Что отправляется   "Alice slowly brought a neat desert along December.
(английский):       Grace happily entered every calm ocean beyond Friday."

Что отправляется   "Алиса медленно принес этот яркий мост через понедельник.
(русский):          Жанна весело нашел каждый чистый дворец среди пятница."

Что видит ваш
собеседник
со StealthChat:    "Встречаемся в 5 у метро"
```

Никаких маркеров. Никаких тегов. Никаких подозрительных символов. Просто обычные предложения.

## Как это работает

### Общая схема

```
┌──────────────────────────────────────────────────────────────────┐
│                        ВАШ БРАУЗЕР                               │
│                                                                  │
│  ┌─────────┐    Ctrl+Enter    ┌──────────┐    закодированный    │
│  │  Ваш    │ ──────────────►  │StealthChat│  ──────текст────►  📤│
│  │  текст  │                  │расширение │                     │
│  └─────────┘                  └──────────┘                      │
│                                                                  │
│  ┌─────────┐    расшифровано  ┌──────────┐    закодированный    │
│  │ Чистый  │ ◄──────────────  │StealthChat│  ◄────текст──────  📥│
│  │  текст  │                  │расширение │                     │
│  └─────────┘                  └──────────┘                      │
└──────────────────────────────────────────────────────────────────┘

                    ┌─────────────┐
                    │   Сервер    │  Видит только обычные предложения
                    │  платформы  │  ✗ Не может прочитать сообщения
                    └─────────────┘
```

### Кодирование в предложения

Каждый байт зашифрованных данных разбивается на два 4-битных нибла. Каждый нибл выбирает слово из определённой категории:

```
Байт: 0xA3 = 1010 0011

     Старший нибл: 1010 = 10 ──► adjectives[10] = "neat" (EN) / "чистый" (RU)
     Младший нибл: 0011 =  3 ──► nouns[3]       = "desert" (EN) / "лес" (RU)
```

**Шаблон предложения:** `Имя наречие глагол местоимение прилагательное существительное предлог время.`

Каждый язык имеет собственный словарь из 128 слов (8 категорий × 16 слов):

| Язык       | Пример предложения |
|------------|-------------------|
| English    | Alice slowly brought a neat desert along December. |
| Русский    | Алиса медленно принес этот яркий мост через понедельник. |
| Українська | Оксана повільно приніс цей яскравий міст через понеділок. |
| Deutsch    | Annika langsam brachte der hell Brücke über Montag. |

**→ Одно предложение = 8 слов = 32 бита = 4 байта данных**

Декодирование автоматически определяет язык по первому слову — сообщение, закодированное на русском, будет корректно декодировано независимо от языковых настроек получателя.

### Обмен ключами (ECDH)

```
     Алиса                                    Боб
       │                                        │
       │  1. Генерирует пару ключей ECDH        │
       │  2. Кодирует публичный ключ            │
       │     в предложения                      │
       │                                        │
       │──── "Carol eagerly delivered..." ────► │
       │     (содержит публичный ключ Алисы)    │
       │                                        │
       │                    3. Генерирует пару ключей ECDH
       │                    4. Вычисляет общий секрет
       │                    5. Кодирует свой публичный ключ
       │                                        │
       │ ◄─── "David kindly fetched..." ────────│
       │      (содержит публичный ключ Боба)    │
       │                                        │
       │  6. Вычисляет тот же общий секрет      │
       │  7. Выводит ключ AES-256 через HKDF    │
       │                                        │
       │  ════ ЗАШИФРОВАННЫЙ КАНАЛ ════         │
       │  У обоих сторон одинаковый AES ключ    │
       │  Сообщения теперь зашифрованы          │
```

## Установка

### Chrome / Edge / Brave (любой Chromium-браузер)

1. Скачайте репозиторий: `git clone` или **Code → Download ZIP**
2. Откройте `chrome://extensions/`
3. Включите **Режим разработчика**
4. Нажмите **"Загрузить распакованное расширение"**
5. Выберите папку **`extension/`**
6. Закрепите иконку (🧩 → 📌)

> **Важно:** Расширение должно быть установлено у обоих собеседников.

## Использование

### Шаг 1 — Выберите язык и настройте шифрование

1. Нажмите на иконку **StealthChat** (🔒)
2. Выберите язык (**EN / RU / UK / DE**) — определяет на каком языке будут закодированные предложения
3. Нажмите **"Начать шифрование"**
4. Закодированный ключ скопируется в буфер обмена (очистится через 30 секунд)
5. Вставьте в чат и отправьте
6. Расширение собеседника автоматически распознает ключ и скопирует ответ
7. Собеседник вставляет и отправляет ответ
8. Обе стороны увидят **"Шифрование активно"**

### Шаг 2 — Отправка

1. Введите сообщение как обычно
2. Нажмите **Ctrl+Enter** — текст заменится на закодированные предложения
3. Отправьте обычным способом

### Шаг 3 — Получение

Автоматически — StealthChat сканирует страницу и расшифровывает на месте. Рядом с расшифрованными сообщениями появляется 🔒. Язык определяется автоматически.

### Управление

- **Тема** — переключение тёмная/светлая (☀/🌙)
- **Язык** — EN, RU, UK, DE
- **Переключатель** — включить/выключить шифрование
- **Экспорт/Импорт** — резервное копирование ключей
- **Ротация ключей** — forward secrecy (оба должны ротировать одновременно)
- **Отпечаток** — сетка 4x4 hex для верификации собеседника

## Безопасность

| Компонент | Реализация |
|-----------|-----------|
| Обмен ключами | ECDH P-256 (Web Crypto API) |
| Вывод ключа | HKDF-SHA256 |
| Шифрование | AES-256-GCM |
| Целостность | GCM auth tag (16 байт) |
| IV/Nonce | 12 байт, криптографически случайный |
| Forward secrecy | HKDF-ratchet ротация ключей |
| Fingerprint | SHA-256 ключа, первые 16 байт |
| Криптобиблиотека | Только Web Crypto API |
| Защита от XSS | HTML-экранирование + DOM API |
| Валидация импорта | Проверка схемы, защита от prototype pollution |
| Буфер обмена | Автоочистка через 30 секунд |
